## 📊 The evaluation dataset (`ragas_eval_dataset.csv`)

A **hand-validated** dataset of 79 question–answer pairs derived from the Moroccan Official Gazette (*Bulletin Officiel*), covering three legal domains:

- **Droit social** — labour and welfare law
- **Droit fiscal** — tax law
- **Droit des sociétés** — corporate law

The dataset includes three question profiles:

| Profile | Description |
|---|---|
| Factual (single-article) | Answer is localised in a single identifiable article |
| Multi-article synthesis | Answer requires combining several fragments from the same instrument |
| Out-of-corpus | Tests the system's ability to recognise the absence of information |

### Columns

| Column | Description |
|---|---|
| `context` | Source passage (chunk) from which the question was generated |
| `question` | Question in natural French |
| `ground_truth` | Reference answer (validated by a human reviewer) |
| `source_doc` | BO issue number |
| `source_article` | Article reference |
| `chunk_index` | Position in the segmented corpus |
| `official_date` | Publication date of the source document |

### Construction protocol

1. **Seed extraction** — chunks of 250–3 000 characters extracted from the indexed corpus
2. **Synthetic generation** — Q/A pairs generated by `gpt-4o-mini` (a model **distinct** from the ones being evaluated, to avoid self-evaluation bias)
3. **Human validation** — every pair was manually reviewed and filtered for correctness, scope and absence of duplicates

This protocol follows the methodological recommendations of Es et al. (2023) on RAG evaluation.

## 📈 Results (`results/`)

Each CSV in `results/` corresponds to one experimental configuration, with the filename encoding the parameters:

```
results_emb={EMBEDDING}_gen={GENERATOR}_hyde={BOOL}_rerank={BOOL}_contextualize={BOOL}_threshold={FLOAT}.csv
```

Each row corresponds to one question, with RAGAS metrics columns:

- `faithfulness` — degree to which the answer is grounded in retrieved context
- `answer_relevancy` — alignment between question and answer
- `context_precision` — proportion of relevant passages in those retrieved
- `context_recall` — proportion of reference content captured by retrieval

The judge model used for RAGAS scoring is `gpt-4o-mini` (kept distinct from the generators evaluated).

## 🚫 What is NOT published

The **seed dataset** (`ragas_seed_dataset.csv`, ~6 MB) is an intermediate file produced by step 1 of the construction protocol. It contains raw extracted chunks before the synthetic generation step. It is excluded from the public release because it is large, derived (re-generatable from the indexed corpus) and not part of the final evaluation artefact.

The **raw PDF corpus** (`data/*.pdf`, ~13 GB) is also excluded for size reasons. The corpus is **publicly available** at <https://www.sgg.gov.ma/> and can be re-crawled using the [`SGGCrawler`](../src/legal_ai/crawlers/sgg_crawler.py) included in this repository.

## 🔁 Reproducing the results

```bash
# 1. Re-crawl the corpus from sgg.gov.ma
python -m legal_ai.main ingest

# 2. Process PDFs into Markdown
python -m legal_ai.main process

# 3. Generate embeddings
python -m legal_ai.main embed

# 4. Run the full evaluation grid (12 configurations × 79 questions)
python -m legal_ai.main eval-all
```

See [docs/EVALUATION.md](../docs/EVALUATION.md) for the full methodology.
